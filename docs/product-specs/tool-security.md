# Tool Security

## User Story

As a LoongClaw operator, I want fine-grained control over which shell commands and
file paths agents can access, so that agents run with least-privilege and cannot
perform destructive or unauthorized operations without explicit configuration.

## Acceptance Criteria

### Shell execution policy

- [ ] Shell command policy is configured entirely via `config.toml`; no rules are
      hardcoded in the binary.
- [ ] `[tools] shell_default_mode` accepts `"deny"` (default) or `"allow"`.
      In `deny` mode, any command not in `shell_allow` is blocked.
- [ ] `[tools] shell_allow` is a list of command basenames that are unconditionally
      permitted. Defaults to `["echo", "ls", "git", "cargo"]`.
- [ ] `[tools] shell_deny` is a list of command basenames that are hard-blocked
      regardless of any other list.
- [ ] `[tools] shell_approval_required` is a list of command basenames that require
      explicit user approval before execution.
- [ ] Policy evaluation order: `shell_deny` → `shell_approval_required` → `shell_allow`
      → `shell_default_mode` fallback.
- [ ] Both bare names (`rm`) and absolute paths (`/usr/bin/rm`, `C:\Windows\...\rm.exe`)
      are matched by extracting the basename.
- [ ] Command matching is case-insensitive.

### File access policy

- [ ] `file.read` and `file.write` tool calls require the `FilesystemRead` /
      `FilesystemWrite` capability on the agent token respectively.
- [ ] `claw.import` requires `FilesystemRead`.
- [ ] When `[tools] file_root` is set, all file paths are resolved relative to that
      root and access outside the root is denied — including paths that traverse `..`
      or follow symlinks to external locations.
- [ ] When `file_root` is unset, the process working directory is used as the root.

### Configuration reference

```toml
[tools]
# Shell execution policy
shell_default_mode = "deny"          # "deny" | "allow"
shell_allow = ["echo", "ls", "git", "cargo"]
shell_deny = []
shell_approval_required = []

# File access sandbox
# file_root = "/path/to/workspace"   # defaults to CWD when omitted
```

### Configuration notes

- The default `config.toml` generated by `loongclaw setup` includes the `[tools]`
  section with the default allow list.
- If a `config.toml` was generated by an older binary without the `[tools]` section,
  the defaults apply automatically (default-deny, allow list as above).
- To regenerate the template with current defaults, run `loongclaw setup --force`
  (this overwrites the existing file).

## Out of Scope

- Argument-level inspection of shell commands (e.g. blocking `git push --force`)
- Interactive approval UI — `shell_approval_required` currently behaves as a soft deny
  until a UI surface is implemented
- Network egress policy
- Process isolation / sandboxing at the OS level
