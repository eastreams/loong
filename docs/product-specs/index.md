# Product Specs

User-facing product requirements and specifications for LoongClaw.

## Structure

Product specs describe **what** the product does from the user's perspective, not **how** it's implemented (that's `design-docs/`) or **when** it's built (that's `exec-plans/`).

## Specs

No product specs yet. Add them here as LoongClaw's user-facing surface grows.

Template for new specs:

```markdown
# [Feature Name]

## User Story
As a [role], I want [capability] so that [benefit].

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2

## Out of Scope
- Item 1
```
