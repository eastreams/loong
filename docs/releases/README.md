# Release Docs Convention

Each released version in `CHANGELOG.md` (for example `## [0.1.0]`) must map to a release process document:

- `docs/releases/v0.1.0.md`

Required content in each release document:

- `# Release vX.Y.Z`
- `## Process`
- `## Detail Links`

`## Detail Links` must include markdown links to concrete release evidence (workflow runs, release page, PR/commits, validation artifacts).

Start from `docs/releases/TEMPLATE.md` when preparing a release.
